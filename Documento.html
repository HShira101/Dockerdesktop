2. Instalación

2.1 Requisitos — Windows

  • Docker Desktop instalado y en ejecución
  • Git

2.2 Requisitos — Linux / Raspberry Pi

  • Docker Engine (docker y docker compose)
  • Git
  • Usuario con permisos sobre el socket Docker (grupo docker)

2.3 Pasos — Windows (desarrollo)

git clone https://github.com/HShira101/Dockerdesktop.git
cd Dockerdesktop
docker compose up --build

Acceder en http://localhost:10000.

El socket de Docker en Windows es //./pipe/docker_engine — ajustar en Compose.yml si se necesita acceso al daemon.

2.4 Pasos — Linux / Raspberry Pi (producción)

git clone https://github.com/HShira101/Dockerdesktop.git
cd Dockerdesktop

Editar Compose.yml: comentar el volumen de desarrollo y descomentar el socket:

# - .:/var/www/html
- /var/run/docker.sock:/var/run/docker.sock
docker compose up --build -d

Acceder desde la red local en http://<IP-RASPBERRY>:10000.

3. Funcionalidad

¿Cómo funciona?

Dockerdesktop es una interfaz web ligera desplegada en un contenedor Docker sobre una Raspberry Pi. Permite visualizar en tiempo real los contenedores Docker que están corriendo en el host, accediendo desde cualquier dispositivo de la red local.

Flujo de datos

  1. El usuario accede a IP-RASPBERRY:10000 desde el navegador.
  2. Laravel recibe la petición en la ruta GET /.
  3. DockerController ejecuta docker ps -a --format "json" usando el socket montado.
  4. Parsea la respuesta línea a línea (cada línea es un objeto JSON independiente).
  5. Devuelve a la vista: nombre, estado e id de cada contenedor.
